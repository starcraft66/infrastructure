= Patroni PostgreSQL HA + Pocket ID Deployment Guide
:toc:
:toclevels: 2

This document describes all manual steps required to deploy the Patroni PostgreSQL cluster and Pocket ID OIDC provider. It covers both:

* **k8s-235-1** -- 3-node HA cluster (sassaflash, stormfeather, soarin)
* **k8s-305-1700-1** -- single-node cluster (spike)

The steps are the same for both clusters unless noted otherwise. Cluster-specific values are called out inline.

== Prerequisites

* Access to the Vault root token (or sufficient privileges)
* YubiKey with the SOPS PGP key for encrypting secrets
* Cloudflare API token available (already configured for ACME)
* `nix develop` shell active (provides `vault`, `sops`, `tofu`, `colmena` CLIs)

== Step 1: Apply DNS records

The DNS records are defined in `dns/dns_tdude.co.tf`.

[cols="20,30,30"]
|===
| Cluster | Pocket ID domain | PostgreSQL domain

| k8s-235-1 | `id.235.tdude.co` | `pg.235.tdude.co`
| k8s-305-1700-1 | `id.305-1700.tdude.co` | `pg.305-1700.tdude.co`
|===

[source,console]
----
cd dns/
tofu init   # only needed on first run
tofu plan   # review changes -- should show 4 new records per cluster (A + AAAA for each)
tofu apply
----

Verify (substitute your cluster's domains):

[source,console]
----
# k8s-235-1
dig +short id.235.tdude.co A         # 172.16.29.11
dig +short id.235.tdude.co AAAA      # 2a10:4741:36:29::11:1
dig +short pg.235.tdude.co A         # 172.16.29.10
dig +short pg.235.tdude.co AAAA      # 2a10:4741:36:29::10:1

# k8s-305-1700-1
dig +short id.305-1700.tdude.co A    # 172.17.51.11
dig +short id.305-1700.tdude.co AAAA # 2a0c:9a46:637:51::11:1
dig +short pg.305-1700.tdude.co A    # 172.17.51.10
dig +short pg.305-1700.tdude.co AAAA # 2a0c:9a46:637:51::10:1
----

== Step 2: Apply Vault Terraform (Patroni AppRole)

The Patroni AppRole is defined in `vault/terraform/modules/vault_k8s/etcd.tf` as `vault_approle_auth_backend_role.patroni`. It grants the `etcd-client-issue` policy which allows issuing etcd client certificates.

[source,console]
----
# For k8s-235-1:
cd vault/terraform/live/235/

# For k8s-305-1700-1:
cd vault/terraform/live/305-1700/

tofu init
tofu plan   # should show 1 new resource: vault_approle_auth_backend_role.patroni
tofu apply
----

== Step 3: Create SOPS-encrypted secret files

Generate the passwords and encryption keys, then create sops-encrypted YAML files. Both Patroni and Pocket ID secrets use a single `environment` key containing `KEY=VALUE` pairs.

Replace `<SITE>` with `235` or `305-1700` and `<PG_DOMAIN>` with `pg.235.tdude.co` or `pg.305-1700.tdude.co`.

=== Patroni secrets

[source,console]
----
SITE=235  # or 305-1700

# Generate strong passwords
REPLICATION_PW=$(openssl rand -base64 32)
SUPERUSER_PW=$(openssl rand -base64 32)
REWIND_PW=$(openssl rand -base64 32)

# Create the plaintext YAML (single environment key with KEY=VALUE pairs)
cat > secrets/patroni-${SITE}.yaml <<EOF
environment: |
  PATRONI_REPLICATION_PASSWORD=${REPLICATION_PW}
  PATRONI_SUPERUSER_PASSWORD=${SUPERUSER_PW}
  PATRONI_REWIND_PASSWORD=${REWIND_PW}
EOF

# Encrypt with sops (must match .sops.yaml creation rule for secrets/patroni-*.yaml)
sops -i -e secrets/patroni-${SITE}.yaml
----

=== Pocket ID secrets

The environment file must contain:

* `DB_CONNECTION_STRING` -- full PostgreSQL connection string including password
* `ENCRYPTION_KEY` -- 32-byte hex key for encrypting OIDC private keys

[source,console]
----
SITE=235  # or 305-1700
PG_DOMAIN=pg.${SITE}.tdude.co

# Generate encryption key
ENCRYPTION_KEY=$(openssl rand -hex 32)

# Generate a password for the pocketid database user
POCKETID_DB_PW=$(openssl rand -base64 32)

# Create the plaintext YAML
cat > secrets/pocket-id-${SITE}.yaml <<EOF
environment: |
  DB_CONNECTION_STRING=postgres://pocketid:${POCKETID_DB_PW}@${PG_DOMAIN}:5432/pocketid
  ENCRYPTION_KEY=${ENCRYPTION_KEY}
EOF

# Encrypt with sops
sops -i -e secrets/pocket-id-${SITE}.yaml
----

IMPORTANT: Note down `POCKETID_DB_PW` -- you will need it in <<Step 8>> to create the database user.

== Step 4: Provision Patroni vault-agent credentials

First, obtain the role ID and generate secret IDs from your workstation.

=== k8s-235-1 (3 nodes)

[source,console]
----
vault login  # with root token or sufficient privileges

CLUSTER_NAME=k8s-235-1
patroni_id=$(vault read --field=role_id auth/approle/role/$CLUSTER_NAME-node-patroni/role-id)

# Generate one secret_id per node
patroni_secret_1=$(vault write --field=secret_id -f auth/approle/role/$CLUSTER_NAME-node-patroni/secret-id)
patroni_secret_2=$(vault write --field=secret_id -f auth/approle/role/$CLUSTER_NAME-node-patroni/secret-id)
patroni_secret_3=$(vault write --field=secret_id -f auth/approle/role/$CLUSTER_NAME-node-patroni/secret-id)

rm ~/.vault-token

# Create directories on all nodes
colmena exec --on=@k8s-235 -- mkdir -p /var/lib/secrets/patroni

# Deploy role_id (same for all nodes)
colmena exec --on=@k8s-235 -- bash -c "echo '${patroni_id}' > /var/lib/secrets/patroni/vault_agent_role_id"

# Deploy secret_id (unique per node)
colmena exec --on=sassaflash -- bash -c "echo '${patroni_secret_1}' > /var/lib/secrets/patroni/vault_agent_secret_id"
colmena exec --on=stormfeather -- bash -c "echo '${patroni_secret_2}' > /var/lib/secrets/patroni/vault_agent_secret_id"
colmena exec --on=soarin -- bash -c "echo '${patroni_secret_3}' > /var/lib/secrets/patroni/vault_agent_secret_id"
----

NOTE: The `role_id` is the same for all 3 nodes, but you *must* generate a separate `secret_id` for each node.

=== k8s-305-1700-1 (single node)

[source,console]
----
vault login

CLUSTER_NAME=k8s-305-1700-1
patroni_id=$(vault read --field=role_id auth/approle/role/$CLUSTER_NAME-node-patroni/role-id)
patroni_secret=$(vault write --field=secret_id -f auth/approle/role/$CLUSTER_NAME-node-patroni/secret-id)

rm ~/.vault-token

colmena exec --on=spike -- mkdir -p /var/lib/secrets/patroni
colmena exec --on=spike -- bash -c "echo '${patroni_id}' > /var/lib/secrets/patroni/vault_agent_role_id"
colmena exec --on=spike -- bash -c "echo '${patroni_secret}' > /var/lib/secrets/patroni/vault_agent_secret_id"
----

== Step 5: Deploy secrets via Colmena

Edit each host's `keys.nix` to set `deploy-keys = true`, then deploy:

[source,console]
----
# k8s-235-1:
# In nixos/hosts/{sassaflash,stormfeather,soarin}/keys.nix, set deploy-keys = true;
colmena apply --on=@k8s-235

# k8s-305-1700-1:
# In nixos/hosts/spike/keys.nix, set deploy-keys = true;
colmena apply --on=spike
----

This deploys the Patroni environment file and Pocket ID environment file to each node. Revert `deploy-keys` back to `false` after deployment.

== Step 6: Deploy NixOS configuration

Deploy the NixOS configuration which activates the Patroni, Pocket ID, HAProxy, and Keepalived services:

=== k8s-235-1

[source,console]
----
colmena apply --on=@k8s-235

# Wait for Patroni to initialize (~30 seconds), then check status
colmena exec --on=sassaflash -- patronictl -c /etc/patroni-pg-235-1-sassaflash.yaml list
# Should show: 3 members - 1 Leader + 1 Sync Standby + 1 Replica
----

=== k8s-305-1700-1

[source,console]
----
colmena apply --on=spike

# Wait for Patroni to initialize (~30 seconds), then check status
colmena exec --on=spike -- patronictl -c /etc/patroni-pg-305-1700-1-spike.yaml list
# Should show: 1 member - Leader
----

=== Keepalived VIPs

colmena exec --on=@k8s-235 -- ip addr show | grep "172.16.29.1[01]"
# Should see 172.16.29.10 (PostgreSQL) and 172.16.29.11 (Pocket ID) on one of the nodes

# k8s-305-1700-1
colmena exec --on=spike -- ip addr show | grep "172.17.51.1[01]"
# Should see 172.17.51.10 (PostgreSQL) and 172.17.51.11 (Pocket ID) on spike
----

=== HAProxy health checks

[source,console]
----
# k8s-235-1 -- Patroni REST API (returns 200 on leader only)
curl -s http://sassaflash.235.tdude.co:8008/primary
curl -s http://stormfeather.235.tdude.co:8008/primary
curl -s http://soarin.235.tdude.co:8008/primary

# k8s-305-1700-1
curl -s http://spike.305-1700.tdude.co:8008/primary

# PostgreSQL via VIP (substitute your PG domain)
psql -h pg.235.tdude.co -U postgres -c "SELECT pg_is_in_recovery();"
# Should return 'f' (false = leader)

# Replicas via VIP port 5433 (k8s-235-1 only -- no replicas on single-node)
psql -h pg.235.tdude.co -p 5433 -U postgres -c "SELECT pg_is_in_recovery();"
# Should return 't' (true = replica)
----

=== ACME certificate

[source,console]

colmena exec --on=spike -- ls -la /var/lib/acme/pocket-id/

== Step 8: Bootstrap Pocket ID database

Connect to the PostgreSQL leader via the local Unix socket and create the database.

=== k8s-235-1

[source,console]
----
# Find which node is the Patroni leader
colmena exec --on=sassaflash -- patronictl -c /etc/patroni-pg-235-1-sassaflash.yaml list

# On the leader node, create the database and user (using trust auth via Unix socket)
colmena exec --on=<leader-node> -- sudo -u patroni psql -h /run/postgresql -U postgres -c "CREATE USER pocketid WITH PASSWORD '<POCKETID_DB_PW from Step 3>';"
colmena exec --on=<leader-node> -- sudo -u patroni psql -h /run/postgresql -U postgres -c "CREATE DATABASE pocketid OWNER pocketid;"

# Restart Pocket ID on all nodes
colmena exec --on=@k8s-235 -- systemctl restart pocket-id
----

=== k8s-305-1700-1

[source,console]
----
# spike is always the leader (single node)
colmena exec --on=spike -- sudo -u patroni psql -h /run/postgresql -U postgres -c "CREATE USER pocketid WITH PASSWORD '<POCKETID_DB_PW from Step 3>';"
colmena exec --on=spike -- sudo -u patroni psql -h /run/postgresql -U postgres -c "CREATE DATABASE pocketid OWNER pocketid;"

# Restart Pocket ID
colmena exec --on=spike -- systemctl restart pocket-id
----

=== Single-node notes

* Patroni runs as a single-member cluster with `synchronous_mode = false`.
* No automatic failover -- if spike goes down, everything is offline.
* HAProxy replicas listener (`port 5433`) will have no healthy backends (the sole node is always leader).
* If nodes are added later, add them to `clusterMembers` in `k8s-305-1700-1/default.nix` and set `synchronousMode = true`.

|===

=== SOPS secret file format

Both `secrets/patroni-<SITE>.yaml` and `secrets/pocket-id-<SITE>.yaml` use a single `environment` key:

[source,yaml]
----
# secrets/patroni-<SITE>.yaml (before encryption)
environment: |
  PATRONI_REPLICATION_PASSWORD=<password>
  PATRONI_SUPERUSER_PASSWORD=<password>
  PATRONI_REWIND_PASSWORD=<password>

# secrets/pocket-id-<SITE>.yaml (before encryption)
environment: |
  DB_CONNECTION_STRING=postgres://pocketid:<password>@pg.<SITE>.tdude.co:5432/pocketid
  ENCRYPTION_KEY=<hex-key>
----
